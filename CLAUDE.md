# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 语言要求

永远只使用中文与用户对话。所有回复、解释、提问、总结等均必须使用中文。

## 仓库用途

这是一个交互式学习仓库。每个课题是一个独立的文件夹。

## 核心规则

### 1. 交互式学习工作流

每个课题的文件夹内，文章按"书名-序号.md"格式命名（如 国富论-1.md、国富论-2.md等）；

用户阅读文章后，需在文章末尾（或文件夹内）写下问题、感悟、反馈；

生成下一篇文章前，会先阅读用户在前一轮的所有反馈，基于反馈调整内容方向；

目标是形成自适应学习模式，确保内容"不过于简单也不过于跳跃"。

### 2. 文章格式约定

文件名：书名-序号.md（如 国富论-1.md）；
文章末尾留出"学习反馈"区域，用 ---分隔，方便用户填写；
行文风格：清晰、有深度，避免空洞感，像"懂行的朋友在讲解"。

## 学习方法论

### 基于 TWO Sigma 研究的核心原则

研究发现：**主动测试比被动阅读的学习效果好 2 个标准差**。

为实现真正的自适应学习，本仓库采用以下机制：

### 机制一：每文主动检索区

每篇文章末尾增加"掌握度自测"区域，内容如下：

```markdown
## 掌握度自测

学完本文后，请尝试回答以下问题（不许翻看原文，凭记忆回答）：

1. **理解度**：用自己的话解释文中核心概念，不讲原文原句。

2. **记忆度**：写出文中三个关键观点（不许翻看）。

3. **应用度**：用文中理论分析一个现实案例。

---
*自评打分（1-5分）：*
- 问题1：_____分
- 问题2：_____分
- 问题3：_____分
- 综合掌握度：_____分（三题平均）
```

### 机制二：三维掌握度评估框架

```
掌握度 = f(理解度, 记忆度, 应用度)

理解度：能用自己的话解释概念（不翻看能讲出来）
记忆度：间隔后还能回忆关键内容（不遗忘）
应用度：能用概念分析现实问题（做对题/分析对案例）
```

| 等级 | 理解度 | 记忆度 | 应用度 | 状态描述 |
|------|--------|--------|--------|----------|
| 5分 | 完全理解，能举例 | 两周不忘 | 随意出题都对 | 融会贯通 |
| 4分 | 理解，稍需提示 | 一周不忘 | 常规题能做对 | 掌握 |
| 3分 | 基本理解，有小疑问 | 三天内不忘 | 变式题稍吃力 | 理解中 |
| 2分 | 似懂非懂 | 第二天就忘 | 照搬原题都难 | 模糊 |
| 1分 | 完全不懂 | 当场就忘 | 不知道怎么入手 | 未掌握 |

### 机制三：掌握度追踪文件

在每个课题文件夹内新建 `掌握度追踪.md`，模板如下：

```markdown
# 《课题名》掌握度追踪

## 核心概念掌握度

| 概念 | 文章 | 理解 | 记忆 | 应用 | 综合 | 状态 |
|------|------|------|------|------|------|------|
| （核心概念） | 章节-序号 | 4 | 3 | 4 | 3.7 | 🟢 |
| （核心概念） | 章节-序号 | 3 | 2 | 2 | 2.3 | 🟡 |

> 状态说明：🟢掌握（≥3.5）🟡理解中（2.5-3.4）🔴模糊（<2.5）

## 遗忘曲线复习计划

基于艾宾浩斯遗忘曲线，关键节点：
- **20分钟后**：简短回顾核心定义
- **1天后**：做一道概念题
- **3天后**：用自己的话复述核心观点
- **7天后**：关联不同章节的概念
- **14天后**：应用分析现实案例

## 待强化概念（综合掌握度 < 3.5）

- [ ] 概念A：下次重点
- [ ] 概念B：需要补充案例

## 学习数据

- 开始时间：
- 文章完成数：
- 平均掌握度：
- 已掌握概念数：
```

## 自适应调整规则

生成新文章时：

1. **读取追踪文件**：查看用户已掌握的概念和薄弱环节
2. **基于反馈调整**：
   - 掌握度低的概念 → 下篇文章重点补充案例和解释
   - 掌握度高的概念 → 快速回顾后引入新内容
   - 用户特别感兴趣的点 → 深入展开
3. **控制节奏**：确保内容"不过于简单也不过于跳跃"
   - 新内容与已掌握内容比例约 7:3
   - 每个新概念至少配一个现实案例
