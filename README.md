# 交互式自适应学习仓库

本仓库是一个**人机协作的主动学习系统**，由 Claude 担任私人导师，帮你真正读懂一本书或一个课题——而不只是"看过了"。

---

## 为什么要这样学？

### 被动阅读的问题

普通读书方式：读 → 划线 → 合上书 → 两周后基本忘光。

学习科学研究（Testing Effect，Roediger & Karpicke 2006）反复证明：**主动检索**的记忆保留效果显著优于重复阅读。读完之后立刻尝试回忆，比再读一遍的效果好得多——这不是直觉，是实验数据。

### 一对一辅导的价值

Bloom（1984）的研究发现，一对一辅导比传统课堂教学的学习效果高出约 2 个标准差——核心原因是**即时反馈**和**个性化节奏**：老师会根据你的实际掌握情况调整下一步，而不是按固定进度表推进。

本仓库用 Claude 部分复现这一效果：每篇文章都基于你上一轮的真实反馈生成，而不是固定课程表。Claude 还会独立判断你的掌握程度，而不是盲目信任你的自评分数。

## 工作流程

```
你读文章
    ↓
完成文末三道自测题（凭记忆，不翻原文）
    ↓
在反馈区写下问题、困惑或感悟（几句话即可）
    ↓
告诉 Claude「生成下一篇」
    ↓
Claude 读取你的反馈
→ 更新掌握度追踪记录
→ 生成专门为你调整的下一篇
    ↓
循环
```

你只需要做两件事：**认真完成自测** + **写几句反馈**。其余由 Claude 处理。

## 三道自测题的设计逻辑

每篇文章末尾固定有三道题，层次递进：

| 题号 | 层次 | 考察什么 | 为什么这样出 |
|------|------|----------|-------------|
| 题1 | 记忆 | 自由回忆核心概念 | 选择题只需"认出来"，自由回忆才需要真正提取 |
| 题2 | 理解 | 换语言解释 / 识别错误用法 | 能离开原文语境说清楚，才是真理解 |
| 题3 | 应用 | 分析一个陌生的现实情境 | 答案不在原文里，必须自己迁移使用 |

**一个判断题目好坏的标准**：如果没读这篇文章，靠常识能答出来——题出坏了。如果答案可以直接在原文找到——也出坏了。好题的特征是：读了才能答，但答案需要自己组织。

## 掌握度评估

Claude 会根据你的实际回答（而不只是你的自评分数）来判断掌握程度，分三个维度：

| 维度 | 含义 | 高分表现 |
|------|------|----------|
| 理解度 | 能用自己的话解释概念 | 不复述原文，能打比方 |
| 记忆度 | 间隔后还能回忆 | 不看文章能说出关键点 |
| 应用度 | 能用概念分析现实 | 主动举新例，而非照搬书中案例 |

综合掌握度 = 三项平均分（1–5分）：

| 综合分 | 状态 | 标志 | 含义 |
|--------|------|------|------|
| ≥ 3.5 | 掌握 | 🟢 | 可以向前推进 |
| 2.5–3.4 | 理解中 | 🟡 | 下篇会补充巩固 |
| < 2.5 | 模糊 | 🔴 | 下篇重点处理 |

**关于校准**：如果 Claude 发现你的自评分数与实际回答质量明显不符，会在下一篇开头说明差异和原因。这不是否定你，而是帮你看到真实的掌握情况，因为高估自己的掌握度是学习停滞最常见的原因之一。

**关于巩固篇**：如果连续两篇掌握度偏低，Claude 会暂停推进新内容，出一篇专门巩固已学概念的文章，之后再恢复正常节奏。

## 间隔复习

艾宾浩斯遗忘曲线的关键节点及操作方式：

| 时间节点 | 建议动作 | 触发方式 |
|----------|----------|----------|
| 学完当天 | 完成文末自测（已内置在文章中） | 无需额外操作 |
| 第3天 | 用自己的话复述核心概念 | 告诉 Claude：「帮我复习[概念名]，出一道题」 |
| 第7天 | 把几个已学概念串联起来 | 告诉 Claude：「第N篇的[概念A]和[概念B]有什么关系？」 |
| 第14天 | 用所学理论做一次完整的现实分析 | 告诉 Claude：「用[概念]分析[你身边的一个情境]」 |

复习时 Claude 会出一道新题并给出反馈，不会重新讲一遍。复习记录会同步更新到追踪文件。

## 开始使用

**第一次开始一个课题**：

1. 在仓库根目录新建文件夹，命名为课题名（如 `国富论`）
2. 打开 Claude Code，说：「我想开始学《国富论》，帮我生成第一篇」
3. 读完后完成自测，在反馈区写几句
4. 说「生成下一篇」，循环开始

**其他常见操作**：

| 想做什么 | 怎么说 |
|----------|--------|
| 开始新课题 | 「我想开始学《XX》，帮我生成第一篇」 |
| 推进当前课题 | 「生成下一篇」 |
| 复习某个概念 | 「帮我复习[概念名]，出一道应用题」 |
| 重新理解某篇 | 「我想重新看第N篇，[具体哪里没懂]」 |
| 查看掌握情况 | 「列一下我目前的掌握度」 |
| 对校准结果有异议 | 「我觉得[概念]我其实理解了，因为……」 |

## 常见问题

**Q：自测题能翻书答吗？**
不能。翻书答对和不翻书答对是完全不同的掌握程度，翻书只是"认出来"，不翻才是真正的记忆提取。哪怕答得不完整也没关系，残缺的回忆比翻书抄写对记忆的帮助大得多。

**Q：反馈区一定要写吗？**
不是强制的，但越认真效果越好。Claude 读取反馈后会调整内容方向——如果什么都不写，下一篇就是没有个性化调整的通用内容。哪怕只写"第二个概念没太懂"，Claude 也能据此做调整。

**Q：Claude 校准了我的分数，但我觉得我其实掌握了，怎么办？**
直接说出来，解释你的理由，Claude 会重新评估。评分校准不是最终判决，它是一个参考起点，你的解释本身也是掌握度的证据。

**Q：想同时学两个课题可以吗？**
可以，在仓库根目录下建两个文件夹，各自独立。但建议一次专注一个，避免概念混淆。

**Q：课题学完了怎么办？**
告诉 Claude「《XX》已经学完，帮我做一个综合复习」，Claude 会出一组跨篇的综合应用题，并给出最终掌握度总结。
